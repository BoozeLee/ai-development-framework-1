# Contributing to ai-development-framework

Thank you for your interest in contributing to ai-development-framework! This document provides guidelines and information for contributors.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/ai-development-framework.git`
3. Create a feature branch: `git checkout -b feature/amazing-feature`
4. Make your changes
5. Commit your changes: `git commit -m 'Add amazing feature'`
6. Push to your branch: `git push origin feature/amazing-feature`
7. Open a Pull Request

## Development Setup

```bash
# Clone the repository
git clone https://github.com/Bakery-street-projct/ai-development-framework.git
cd ai-development-framework

# Install dependencies
pip install -r requirements.txt

# Run tests
python -m pytest

# Run linting
flake8 .
```

## Code Style

- Follow PEP 8 for Python code
- Use meaningful variable and function names
- Add docstrings to functions and classes
- Write unit tests for new features
- Update documentation as needed

## Pull Request Process

1. Ensure your code follows the project's style guidelines
2. Add tests for new functionality
3. Update documentation if necessary
4. Ensure all tests pass
5. Request review from maintainers

## Issue Reporting

When reporting issues, please include:

- Description of the issue
- Steps to reproduce
- Expected behavior
- Actual behavior
- Environment details (OS, Python version, etc.)

## Security

- Do not include sensitive information in issues or PRs
- Report security vulnerabilities privately
- Follow our [Security Policy](SECURITY.md)
